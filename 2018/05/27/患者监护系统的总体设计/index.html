患者监护系统的总体设计

1. 引言

1.1. 编写目的

  总体设计是概括地说明PMS实现过程。使用需求分析中得到的数据流图进行设计，明确系统中的模块组成，确定程序设计方案，从而明确系统结构。本文档供项目经理、开发者、客户参考。

1.2. 项目背景

  • 项目名称：患者监护系统
  • 委托单位：xx医院
  • 开发单位：xx公司
  • 主管部门：xx部门
  • 目标用户：医生护士及其他工作人员

    1.3. 定义

  • PMS: Patient Monitoring System 患者监护系统
  • SQL Server: 系统所使用的数据库系统
  • PyCharm: 系统开发所用工具

1.4.参考资料

  略

2. 任务概述

2.1. 目标

  划分出组成系统的物理结构，设计系统的总体结构，确定系统模块组成，模块功能及模块和模块之间的接口、调用关系等。

2.2. 运行环境

  该系统为B/S三层结构，它的运行环境应分为客户端、应用服务器端和数据库服务器端三部分。
  以下是系统运行环境：
  （1）客户端
    操作系统：Windows 10
    浏览器：IE11 以上或者 Chrome 63.0.3239.132/更新版本
  （2）应用服务器端
    操作系统：Windows Server 2017 或更新版本
    应用服务器：Tornado 5.0.2 或更新版本
  （3）数据库服务器端
    服务器端：Windows Server 2017 或更新版本
    数据库系统：SQL Server 2017 或更新版本

2.3.需求概述

  PMS为监护患者病情而建立的系统，围绕着患者和护士在护士、患者信息、生理信号等方面进行管理和监测。PMS连接着患者信息数据库和安全信息数据库，并将患者监护设备实时传来的生理信号进行监控和定时记录。若生理信号异常，则将报警及时显示在终端提醒护士采取操作。护士可以要求系统生成某一患者在某一时段的特定病情报告。
