患者监护系统的详细设计

1. 引言

1.1. 编写目的

1.2. 项目背景

  • 项目名称：患者监护系统
  • 委托单位：xx医院
  • 开发单位：xx公司
  • 主管部门：xx部门
  • 目标用户：医生护士及其他工作人员

    1.3. 定义

  • PMS: Patient Monitoring System 患者监护系统
  • SQL Server: 系统所使用的数据库系统
  • PyCharm: 系统开发所用工具

2. 程序描述

2.1. 功能

  • （1） 子模块一：用户登入系统并查看基本信息。
  • （2） 子模块二：患者信息管理，能对患者信息进行增删改查看操作。
  • （3） 子模块三：生理状况监护，能对异常信息及时地发出警报，定时生成患者日志。
  • （4） 子模块四：病情报告，生成并展示某患者的病情报告。

2.2. 性能

  • （1） 子模块一：管理员可以对登陆注册用户进行管理。
  • （2） 子模块二：应能及时反馈存储情况，数据稳定存储。
  • （3） 子模块三：及时分析处理信息并反馈。
  • （4） 子模块四：能够按照用户意愿自由生成报告。

2.3. 输入和输出项目

输入：

患者信息（患者编号[int(8)]、姓名[char(8)]、年龄[int(2)]、入院时间[int(8)]、病房号[int[3]]、病床号[int(2)]、所患病症[char(40)]、备注[char(80)]等）
生理信号（血压[float(5)]、体温[float(5)]、脉搏[float(5)]、心电图[img]等）
护士请求（请求生成病情报告[log]、查看患者信息[json]等）
时钟信息（本地时钟信息[time]）
安全信息（生理信号安全范围[array]）

输出：

警告信息
病情报告
患者日志
