2. 任务概述

2.1. 目标

  本项目名称为“患者监护系统”。目前，住院病人主要由护士护理，这样做不仅需要大量护士，而且由于不能随时观察病人的病情变化，还会延误抢救时间。因此，急需开发一个“患者监护系统”，来减轻工作量并且提高效率，为病人提供更好的服务。该系统功能主要包括：

  • 能够随时接收病人的生理信号（血压、体温、脉搏、心电图等），监视病员的病症
  • 定时记录病人情况以形成患者日志（定时更新病历）
  • 当某个病人的生理信号超出医院的规定的安全范围时向值班护士发出警告信息（病员出现异常情况时报警）
  • 护士在需要时还可要求系统打印某个病人的病情报告（随机地产生某一病员的病情报告）

2.2. 运行环境

  该系统为B/S三层结构，它的运行环境应分为客户端、应用服务器端和数据库服务器端三部分。
  以下是系统运行环境：
  （1）客户端
    操作系统：Windows 10
    浏览器：IE11 以上或者 Chrome 63.0.3239.132/更新版本
  （2）应用服务器端
    操作系统：Windows Server 2017 或更新版本
    应用服务器：Tornado 5.0.2 或更新版本
  （3）数据库服务器端
    服务器端：Windows Server 2017 或更新版本
    数据库系统：SQL Server 2017 或更新版本

2.3. 条件与约束

  （1）当病人的生理信号发生变化时，都应增加相关病人病历库的记录；
  （2）系统的外部项至少包括：病人、护士。
  （3）本系统的预算是xx万元
  （4）本系统预期完成时间xx

3. 数据描述

3.1. 静态数据

  患者姓名、性别、年龄、病房号、病床号、病症

3.2. 动态数据

  患者生理信号（血压、体温、脉搏、心电图等）、时钟信息、患者日志、病情报告

3.4. 数据字典

名字：生理信号安全范围
别名：安全范围
描述：用来表示正常患者的生理情况的正常范围
定义：安全范围=血压+脉搏+体温+心电图（正常范围）
位置：存储在安全范围文本中

名字：患者日志
别名：无
描述：定时记录患者生理信号，并与安全范围进行对比
定义：患者日志=患者编号+血压+脉搏+体温+心电图
位置：存储在患者日志数据库中

名字：患者信息
别名：无
描述：记录患者个人信息
定义：患者信息=患者编号+姓名+性别+年龄+入院时间+病房号+病床号+所患病症
位置：存储在患者信息数据库中
